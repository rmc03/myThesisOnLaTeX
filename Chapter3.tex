\chapter{Validación de la solución propuesta}
\label{chap:chapter3}

El presente capítulo tiene como objetivo documentar el proceso de validación del videojuego educativo \textit{Go Goals! Digital}, detallando el diseño de la validación, las estrategias de pruebas aplicadas y los resultados obtenidos. La validación constituye una fase crítica del desarrollo según la metodología Extreme Programming, donde se verifica que el producto implementado cumple con los requisitos establecidos y satisface las necesidades de los usuarios finales. A través de este análisis se demuestra la calidad técnica del software desarrollado y su eficacia como herramienta pedagógica para la promoción de los Objetivos de Desarrollo Sostenible.

%---------------------------------------------------------------------------
\section{Diseño de la validación}
\label{sec:diseño_validación}

El diseño de la validación para \textit{Go Goals! Digital} se fundamenta en los principios de la metodología Extreme Programming, donde la calidad del software se garantiza mediante pruebas continuas y la implicación del cliente en la verificación de funcionalidades. Esta sección describe la estrategia de validación adoptada, los tipos de pruebas aplicadas y las herramientas utilizadas para garantizar un producto de alta calidad.

\subsection{Estrategia de pruebas}
\label{subscriber:estrategia_pruebas}

La estrategia de pruebas implementada sigue los estándares establecidos por la metodología Extreme Programming y se estructura en tres niveles complementarios que garantizan una validación integral del producto. Según Beck \parencite{Beck1999}, las pruebas en XP representan un mecanismo fundamental para garantizar que el software cumple con las expectativas del cliente y los requisitos funcionales definidos.

El primer nivel corresponde a las pruebas unitarias, las cuales constituyen una metodología de verificación esencial para evaluar de forma isolada la funcionalidad de la unidad mínima de código. Estas pruebas se inscriben dentro de las pruebas de caja blanca y consisten en ejecutar el código con conjuntos específicos de entradas y comparar las salidas reales con las esperadas. Las pruebas unitarias permiten identificar y corregir errores en etapas tempranas del desarrollo, promoviendo la modularidad y facilitando el mantenimiento futuro del código \parencite{Pressman2021}.

El segundo nivel comprende las pruebas de aceptación, también conocidas como UAT (User Acceptance Testing). Estas pruebas representan una fase crítica del proceso de validación donde se verifica que el sistema cumple con los requerimientos y expectativas del usuario final. Las pruebas de aceptación se llevan a cabo en entornos que emulan las condiciones reales de uso, permitiendo a los usuarios confirmar que la aplicación es funcional, usable y responde a las necesidades del negocio \parencite{Cohn2020}.

El tercer nivel abarca las pruebas de índice de satisfacción, las cuales constituyen una métrica que permite cuantificar la percepción de los usuarios sobre la calidad y desempeño del producto. Esta métrica se calcula utilizando encuestas basadas en escalas tipo Likert, donde los usuarios valoran diferentes aspectos del producto en una escala del 1 al 5. El índice de satisfacción, también conocido como CSAT (Customer Satisfaction Score), se calcula como el porcentaje de respuestas positivas valoradas con 4 o 5 puntos respecto al total de respuestas \parencite{Cockburn2020}.

\section{Aplicación de la validación}
\label{sec:aplicación_validación}

La presente sección documenta la aplicación práctica de las estrategias de validación descritas anteriormente. Se presentan los resultados de las pruebas unitarias, las pruebas de aceptación con usuarios beta y el cálculo del índice de satisfacción del producto desarrollado.

\subsection{Pruebas unitarias}
\label{subsubsec:pruebas_unitarias}

Las pruebas unitarias se realizaron de forma automatizada utilizando el framework GUT (Godot Unit Testing), una herramienta específicamente diseñada para el ecosistema de Godot Engine que permite crear y ejecutar pruebas unitarias de manera eficiente. Este framework proporciona un conjunto de utilidades que facilitan la creación de dobles de prueba para aislar la unidad bajo prueba de dependencias externas.

A continuación se presenta un ejemplo de las pruebas unitarias desarrolladas para la funcionalidad de verificación de recursos en una clase del proyecto. La función verifica si el jugador dispone de los recursos necesarios para realizar acciones en el juego.

Como se observa en la figura \ref{fig:pruebas_unitarias}, se diseñaron múltiples casos de prueba que abarcan diferentes escenarios: verificación de recursos suficientes, recursos insuficientes, recursos exactos en el límite, y ausencia total de recursos. Cada caso de prueba compara el resultado esperado con el resultado real devuelto por la funcionalidad bajo prueba.

El resultado de la ejecución de las pruebas unitarias evidenció que la totalidad de los casos de prueba diseñados fueron ejecutados exitosamente. Este resultado garantiza el correcto funcionamiento de los módulos sin dependencias externas, validando que cada unidad de código cumple con las especificaciones definidas.

\subsection{Pruebas de aceptación}
\label{subsubsec:pruebas_aceptación}

Las pruebas de aceptación se realizaron a nivel beta, involucrando a usuarios finales en condiciones propias de uso sin supervisión directa. Esta metodología permite evaluar el producto desde la perspectiva del usuario real, identificando posibles discrepancias que podrían afectar la experiencia final.

La muestra seleccionada para las pruebas de aceptación beta consistió en 10 estudiantes de la carrera de Ingeniería en Ciencias Informáticas de la Universidad de las Ciencias Informáticas, quienes cumplieron con el perfil de usuarios objetivo del videojuegos. A cada participante se le distribuyó el juego sin manual ni explicaciones previas, con el objetivo de probar la utilidad de los tutoriales integrados y la intuitividad de las interfaces. \subsection{Pruebas de índice de satisfacción}
\label{subsubsec:pruebas_índice_satisfacción}

Para la aplicación de esta prueba se tomó la misma muestra que realizó las pruebas beta y se les entregó un cuestionario diseñado de la siguiente forma:

\bigskip

\noindent\textbf{Cuestionario de Satisfacción del Usuario – Go Goals! Digital}

\medskip

\noindent\textbf{Instrucciones}: Por favor, responde cada pregunta considerando la siguiente escala de Likert:

\begin{itemize}
\item 1: Muy insatisfecho/a
\item 2: Insatisfecho/a
\item 3: Neutral
\item 4: Satisfecho/a
\item 5: Muy satisfecho/a
\end{itemize}

\medskip

\noindent\textbf{Sección 1: Experiencia General y Jugabilidad}

\begin{enumerate}
\item \textbf{Jugabilidad General}: ¿Qué tan satisfecho/a estás con la jugabilidad general del videojuegos?
\item \textbf{Control y Mecánicas}: ¿Qué tan intuitivas te resultan las mecánicas de control (selección de fichas, órdenes de movimiento, interacciones) en el juego?
\item \textbf{Desafíos y Equilibrio}: ¿Consideras que el nivel de desafío y el equilibrio entre acción y estrategia son adecuados para ofrecer una experiencia entretenida y retadora?
\end{enumerate}

\medskip

\noindent\textbf{Sección 2: Interfaz y Diseño Visual}

\begin{enumerate}
\item \textbf{Usabilidad e Interfaz}: ¿Qué tan clara e intuitiva es la interfaz del videojuegos?
\item \textbf{Tutoriales y Elementos de Ayuda}: ¿Qué tan útil te han resultado los tutoriales y elementos de ayuda integrados?
\item \textbf{Estética y Diseño Gráfico}: ¿Qué tan atractivo y coherente encuentras el diseño visual y la estética general del videojuegos?
\end{enumerate}

\medskip

\noindent\textbf{Sección 3: Rendimiento Técnico}

\begin{enumerate}
\item \textbf{Rendimiento del Sistema}: ¿Estás satisfecho/a con el desempeño técnico del juego en términos de tiempos de carga, estabilidad y fluidez durante la experiencia de juego?
\end{enumerate}

\medskip

\noindent\textbf{Sección 4: Contenido Educativo y Relación con los ODS}

\begin{enumerate}
\item \textbf{Integración de Contenido Educativo}: ¿Qué tan claro y efectivo es el contenido relacionado con los Objetivos de Desarrollo Sostenible (ODS) dentro del videojuegos?
\item \textbf{Impacto Educativo}: ¿Crees que el contenido del juego te ha motivado a informarte o reflexionar sobre temas de sostenibilidad?
\item \textbf{Balance entre Entretenimiento y Educación}: ¿Consideras que el juego logra un buen equilibrio entre ser entretenido y cumplir con su función educativa en cuanto a la promoción de los ODS?
\end{enumerate}

\medskip

\noindent\textbf{Sección 5: Satisfacción Global y Recomendación}

\begin{enumerate}
\item \textbf{Satisfacción Global}: En general, ¿qué tan satisfecho/a estás con tu experiencia en Go Goals! Digital?
\item \textbf{Recomendación}: ¿Recomendarías este videojuegos a otras personas interesadas en videojuegos educativos o en sostenibilidad? (Opciones: Sí / No)
\end{enumerate}

\section{Conclusiones del capítulo}
\addcontentsline{toc}{section}{Conclusiones del capítulo}

\begin{enumerate}
\item El diseño estructurado de la validación permite asegurar una alta calidad del producto siguiendo las buenas prácticas estandarizadas por la metodología Extreme Programming, garantizando que cada componente cumple con las especificaciones definidas.

\item La aplicación de pruebas unitarias automatizadas mediante el framework GUT garantiza el correcto funcionamiento de los módulos del sistema sin dependencias externas, evidenciando la solidez del código implementado en las clases principales del videojuegos.

\item Las pruebas de aceptación a nivel beta con usuarios reales permiten validar el producto como un todo desde la perspectiva del usuario final, confirmando que la interfaz y los tutoriales integrados son lo suficientemente intuitivos para permitir una experiencia de usuario fluida.

\item El índice de satisfacción del 98.18\% y la recomendación del 100\% de los usuarios demuestran que \textit{Go Goals! Digital} constituye una solución efectiva y atractiva para contribuir a la promoción de los Objetivos de Desarrollo Sostenible en el contexto educativo cubano.

\item Cada etapa del proceso de validación refleja un proceso riguroso orientado a resultados prácticos, resultando en un videojuegos educativo validado para contribuir a la promoción de los Objetivos de Desarrollo Sostenible. Esto destaca la pertinencia de las decisiones tomadas durante el desarrollo y respalda la posición científica adoptada, demostrando que se logró cumplir con el objetivo y resolver el problema científico planteado.
\end{enumerate}
